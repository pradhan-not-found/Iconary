# Starter Icon List (First 30)

Use these to launch Iconary with a solid foundation. Each includes a suggested name, category, and a draft concept explainer (feel free to refine wording).

## Core AI Concepts
1. **ai-agent** — An AI agent is a system that can perceive its environment and take actions to achieve a goal, often using tools or making decisions autonomously.
2. **ai-model** — A model is the trained system that takes inputs and produces outputs, such as predictions or generated content.
3. **ai-prompt** — A prompt is the input text or instructions given to an AI model to guide its response.
4. **ai-token** — A token is a chunk of text (word, part of a word, or character) that a model processes as a basic unit.
5. **ai-hallucination** — A hallucination is when an AI model generates information that sounds plausible but is factually incorrect or fabricated.
6. **ai-context-window** — The context window is the amount of text a model can consider at once when generating a response.
7. **ai-fine-tuning** — Fine-tuning is the process of further training a pre-existing model on specific data to specialize its behavior.
8. **ai-inference** — Inference is the process of running a trained model to produce an output from new input data.
9. **ai-temperature** — Temperature is a setting that controls how random or predictable a model's output is.
10. **ai-multimodal** — Multimodal means a model can understand and generate multiple types of data, such as text, images, and audio.

## Data & Pipelines
11. **data-pipeline** — A data pipeline is a series of steps that move and transform data from one system to another.
12. **data-vector-database** — A vector database stores data as numerical representations (embeddings) for fast similarity search.
13. **data-embedding** — An embedding is a numerical representation of data (like text or images) that captures its meaning for machine processing.
14. **data-labeling** — Data labeling is the process of tagging raw data with meaningful labels for training machine learning models.
15. **data-preprocessing** — Preprocessing is the cleaning and transforming of raw data before it's used to train or run a model.
16. **data-dataset** — A dataset is a structured collection of data used to train, validate, or test a model.
17. **data-batch** — A batch is a group of data samples processed together during training or inference.
18. **data-streaming** — Streaming data refers to data that is generated and processed continuously in real time.

## Architecture & Workflow
19. **workflow-rag** — RAG (Retrieval-Augmented Generation) is a technique where a model retrieves relevant information from external sources before generating a response.
20. **workflow-orchestration** — Orchestration is the coordination of multiple AI components or services to complete a larger task.
21. **workflow-pipeline-step** — A pipeline step is a single stage in a larger automated workflow.
22. **workflow-api** — An API is an interface that lets different software systems communicate with each other.
23. **workflow-webhook** — A webhook is an automated message sent from one system to another when a specific event occurs.

## Infrastructure
24. **infra-gpu** — A GPU (Graphics Processing Unit) is specialized hardware that accelerates the parallel computations needed for training and running AI models.
25. **infra-container** — A container is a lightweight, portable package that bundles software and its dependencies so it runs consistently across environments.
26. **infra-scaling** — Scaling refers to adjusting computing resources up or down to match demand.
27. **infra-monitoring** — Monitoring is the ongoing tracking of system performance and behavior to detect issues.

## Safety & Evaluation
28. **safety-guardrail** — A guardrail is a rule or system designed to keep an AI model's outputs within safe or intended boundaries.
29. **eval-benchmark** — A benchmark is a standardized test used to measure and compare the performance of different models.
30. **eval-metric** — A metric is a measurable value used to evaluate the performance or quality of a model's output.
